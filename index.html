📘 功能使用說明書
1. 建議使用順序
  1. 先到設定建立藝人、通路、專輯類型與批次。
  2. 到「通路」建立通路總訂單與總下單數。
  3. 使用一般新增或快速新增，建立購買人訂單。
  4. 到確認中心核對付款時間、數量與金額。
  5. 完成登記與出貨後，把通路訂單放進完成群組。
2. 通路訂單
  • 通路訂單依「藝人 → 通路 → 批次」分類。
  • 總下單數會自動扣除已登記數量並顯示剩餘。
  • 購買日期可輸入兩筆，例如 0812 0815。
  • 簽售日期可輸入一筆，例如 0820。
  • 「＋ 賣貨便連結」可輸入網址；儲存後可一鍵複製。
  • 「💰 批量改金額」可輸入 1=200 2=350，依數量批量更新金額。
3. 快速新增購買人訂單
  • 「多人同通路」：先選通路，再逐列輸入購買人、數量、金額與備註。
  • 同一次快速新增可再加入第二、第三個通路訂單；每個通路都有獨立日期、購買人欄位與分隔線。
  • 「同一人多通路」：先選購買人，再逐列選擇不同通路。
  • 每個通路區塊可各自設定 1＝200、2＝350；數量改變時只會套用該通路的金額。
  • 通路下拉只顯示尚未登記完成的訂單。
  • 快速新增預設為已付款，建立前可先按「預覽」。
  • 「建立後繼續下一批」會保留尚有數量的通路並清空名單，可接著建立下一批。
  • 購買人欄位可展開完整名單，也可保留搜尋或直接輸入新名字。
4. 購買人訂單
  • 訂單依購買人、藝人、通路與詳細訂單分層收合。
  • 離開訂單頁面後，所有 Tree View 會自動收起。
  • 可切換付款狀態，並勾選建立賣場、下單及出貨。
  • 若姓名建議沒有被選取，系統會帶入第一位符合者並顯示警告符號。
  • 購買日期、簽售日期與賣貨便連結會跟通路訂單同步。
  • 可在頁面下方選取同一人的指定訂單，另外建立「單人多通路訂單」與專屬賣貨便連結。
5. 訂單確認中心
  • 待確認區只列出尚未完成三項確認的訂單。
  • 分別確認付款時間、數量及金額。
  • 確認付款時間時會記錄當下時間並設為已付款。
  • 訂單金額被修改後，金額確認狀態會自動取消。
  • 可輸入姓名、付款時間、數量與金額，由時間優先尋找可能的通路訂單。
  • 三項確認完成後會自動移到可搜尋的確認封存區。
6. 完成專區
  • 通路登記數量已滿，且所有購買人訂單都已出貨後，即符合完成條件。
  • 符合條件但尚未分類時，系統會跳出提醒。
  • 可選擇既有群組，或建立自訂名稱的新群組。
  • 加入群組後，通路訂單狀態會自動改為「已完成」。
7. 備份與資料安全
  • 重要修改前建議先在備份中心建立手動備份。
  • JSON 匯出可作為第二份外部備份。
  • 匯入舊版備份時，系統會自動升級資料格式。
  • 「清空全部資料」無法復原，操作前務必確認已有備份。

修改後會保存在目前網站資料中，日期與訂單統計仍會自動更新。
